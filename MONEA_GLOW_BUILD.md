# MONEA GLOW｜構築指示プロンプト（決定版）

このファイルの本文を、VS Code の Claude Code にそのまま貼ってください。
同じフォルダの `index.html` を**完成デザインの見本**として、見た目・配色・演出・レイアウトを再現します。

---

## 0. サイトのテーマ（最優先・絶対に忘れないこと）

> **自分らしく美しくなりたいすべての人へ。**
> **誰かになるためではなく、自分自身の魅力を見つけ、花開かせるために。**

このテーマがすべての判断基準です。コピー・写真・余白・演出は、この世界観に沿わせること。
「色づく＝自分の魅力が花開く」という比喩を、サイト全体の演出の軸にする。

---

## 1. ブランド（変更しないこと）
- 名称: **MONEA GLOW**
- タグライン: 自分らしい輝きを、毎日に。
- ドメイン: moneaglow.com（Cloudflare 取得済み）
- ターゲット: 自分らしく美しくなりたいすべての人

## 2. 技術構成
- 制作: VS Code + Claude Code、フロントは静的 HTML/CSS/JS（ビルド不要、原則バニラ）
- 公開: GitHub + Vercel（自動デプロイ）、独自ドメイン moneaglow.com を接続
- サーバ処理: Vercel Serverless Functions（`api/` 配下）
- キー・設定はコードに直書きせず、リポジトリ（Vercel）の環境変数に置く

## 3. デザインシステム（白背景モノトーン）
- 配色は**無彩色のみ**: 背景 #FFFFFF / オフホワイト #F7F7F5、文字 #1A1A1A、補助グレー #777775、罫線 rgba(0,0,0,.1)、黒地セクション #1A1A1A
- **金・暖色・その他の有彩色は使わない**
- 写真・画像は**すべてグレースケール**で配置し、インタラクションで色がつく（下記6）
- タイポ: 英字 Cormorant Garamond / 和文見出し Noto Serif JP / 本文 Noto Sans JP
- 装飾よりも、余白・タイポ・白黒のコントラスト・光で見せる

## 4. レイアウト（参考: https://www.chanel.com/jp/）
シャネルのトップに倣った**ミニマルな編集型レイアウト**にする。
- 大きなビジュアルを縦に積む。各ブロックは「カテゴリ名（小）＋短い見出し＋リンク1つ」だけ。テキストは最小、画像が主役
- ヘッダーはミニマル（左メニュー / 中央ロゴ MONEA GLOW / 右に検索・お気に入り・カート）。ヒーロー上では透過、スクロールで白背景
- グリッドに商品を敷き詰めるより、特集ブロックを縦に見せることを優先する

## 5. メインビジュアル（白黒 → 色づく）
- ヒーローは**白黒写真が、光が差すとともに色づいていく**演出（テーマの体現）。
- 本番は同じ構図の**カラー実写**を入れ、grayscale(1) → grayscale(0) で本物の色が花開くようにする。
- カラー素材が未用意のうちは、`index.html` の疑似着色演出（sepia/saturate + カラーオーバーレイ）で代用する。

## 6. インタラクション（触れると色づく → 下層へ）
- 各レイアウトセクション（特集ブロック・カテゴリ）は通常**白黒**で、**ホバー／タップで色づく**（grayscale が解けてカラーになる）。
- 色づいたブロックをクリックすると、そのカテゴリの**下層ページ**へ遷移する。
- 全体は白黒で静かに佇み、触れた所だけ色が灯る——テーマの「魅力が花開く」を操作で体験させる。

## 7. 画像（すべて実写・イラストなし）
- アイコン的イラストや作り物の容器画像は使わない。**すべて実写写真**で構成する。
- 特集・ヒーローの写真は、本番ではカラー実写を用意して差し込む（用意できるまでは見本のプレースホルダで構造を作る）。
- 商品写真はドロップシッピングのサプライヤー画像（実写）を自動取得して使う。

## 8. ページ・セクション
トップ（最優先で完成）: ヘッダー → ヒーロー（白黒→色づく）→ 信頼バー → 特集ブロック（縦積み・ホバーで色づく）→ いま人気の美容アイテム（自動入れ替え）→ コンセプト（テーマ掲示）→ 5分診断への導線 → メンバー登録 → フッター。
下層ページ: about / products / product / category（各カテゴリ）の枠を用意。
- **現段階では、利用規約・特定商取引法に基づく表記・プライバシーポリシー・会社住所などの法務ページ／フッターの法務リンク・住所表記は作らない**（後日追加）。フッターはブランド・商品カテゴリ・SNS・決済表示・コピーライトのみ。

## 9. 機能（すべて実装）
- カート（追加・数量変更・削除・小計）、お気に入り、検索、モバイルメニュー、メルマガ登録
- メンバー登録ポップアップ（下記12）、5分診断モーダル（下記12）
- スクロール出現アニメーション、レスポンシブ（モバイル最適化必須）

## 10. ドロップシッピング：人気アイテムの自動入荷・自動入れ替え
- 並び順は「自社売上」ではなく、外部フィード由来の**人気度（トレンド指数）**。立ち上げ直後・売上ゼロでも商品が並ぶこと。
- 「いま人気の美容アイテム」を一定間隔で再取得し、トレンド商品が**自動で入れ替わる**。順位変動はハイライト。失敗時はモックにフォールバック。
- 各カードに在庫種別（国内発送／海外直送・取り寄せ）とお届け目安を表示。
- **フロント上で「商品が自動で入れ替わる仕組み」だと訪問者に明示する**: 説明コピー（例「人気の美容トレンドに連動して、商品は自動で入れ替わります」）＋自動更新中のライブインジケータ（点滅ドット＋「自動更新中」）。
- 商品は「化粧品（薬機法対象・国内発送）」と「美容雑貨（対象外・海外直送可）」に分け、自動入れ替えの主対象は美容雑貨。化粧品は国内・正規仕入れに限定。知財・商標侵害品は扱わない。

## 11. 主要ドロップシッピングの API 連携の仕組み
- `api/products.js`（Vercel Serverless）が主要ドロップシッピング各社の API を集約し、人気順の商品を共通フォーマット（n / en / p / img / type / trend / lead / supplier / sku / url）で返す。
- 対応プロバイダ: **CJ Dropshipping / AliExpress / Spocket / DSers / Zendrop**。環境変数 `DROPSHIP_PROVIDER` で選択。アダプタ方式なので他社も追加可能。
- API キー・トークンは環境変数に設定（`CJ_ACCESS_TOKEN`, `ALIEXPRESS_APP_KEY` ほか。`api/products.js` 冒頭のコメント参照）。
- フロントは `FEED.useApi=true` のとき `/api/products?sort=best_selling&limit=N` から取得、失敗時はモックにフォールバック。

## 12. 信頼バー・ポップアップ・5分診断
- 信頼バー（2項目）: ①「メンバーはいつでも10%OFF」→メンバー登録へ ②「5分でわかる肌診断」→診断モーダルへ。送料無料・サンプル等は出さない。
- ポップアップは**メンバー登録のアピール**（いつでも全品10%OFF・新作先行案内）。商品オファーにしない。
- 5分診断: MBTI のように簡素な5問。回答からユーザーに最適なカテゴリと商品を提案し、最後にメンバー登録へ誘導。

- 規約・表記で決済代行会社を社名で書かず「当法人が指定する決済代行会社」とする（法務ページは現段階では作らない）。

## 16. 出力・品質ルール
- すべての HTML/CSS/JS は**レスポンシブ・モバイル最適化**必須。

## 17. 構築・公開手順
1. このフォルダを `~/github/v0-monea` に置き、VS Code で開く
2. `index.html` を見本に、トップページから実装（配色・ヒーロー・演出を再現）
3. `api/products.js` を活かして商品フィードを実装（まずはモック、キー設定後に `FEED.useApi=true`）
4. GitHub へ push → Vercel に連携 → moneaglow.com を接続
5. ドロップシッピング各社のキーを Vercel の環境変数に設定

## 19. 設計見本
- `index.html` … 白背景モノトーン＋シャネル風の縦積みレイアウト＋白黒→色づくヒーロー＋ホバーで色づく特集ブロック＋自動入れ替え＋5分診断＋メンバー登録。**配色・演出・構成はこれを基準にそのまま再現**すること。
- `category.html` … カテゴリ下層ページ（`?cat=` でカテゴリ切替、フィルタUI、商品グリッド）。
- `diagnosis.html` … 5分診断の専用フォームページ（5問・進捗・結果・おすすめ・メンバー導線）。
- `media.html` / `article.html` … メディア（記事一覧／個別記事）。

## 20. メディア（SEO記事）と自動発信
- `media.html`（記事一覧）と `article.html`（個別記事）。個別記事は `<title>`／メタディスクリプションを記事に合わせて動的更新し、JSON-LD（Article）を出力してSEOに対応する。
- `api/generate-article.js` … Claude API（Anthropic Messages）で美容記事を自動生成し、Vercel KV に保存する。`vercel.json` の cron で定期実行（既定: 毎日23:00 UTC）。テーマはローテーション。**薬機法に配慮**し、化粧品で認められた範囲を超える効能・効果の断定表現を避けるようシステムプロンプトで制御。
- `api/articles.js` … 保存記事を一覧／個別で返す。フロントは `FEED.useApi=true` で `/api/articles` を参照し、失敗時はページ内モック記事にフォールバック。
- `api/sitemap.js` … 記事URLを含む `sitemap.xml` を動的生成（`/sitemap.xml` に rewrite）。`robots.txt` から参照。
- 環境変数（Vercel）: `ANTHROPIC_API_KEY` / `ANTHROPIC_MODEL`（既定 claude-sonnet-4-6）/ `CRON_SECRET`（任意）/ `SITE_URL` / `SITE_NAME`、および Vercel KV 接続。
- 依存: `@vercel/kv`（`package.json`）。デプロイ前に Vercel KV を作成・接続すること。
